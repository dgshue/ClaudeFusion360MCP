    **Rewrite fusion-addin/FusionMCP.py:**
    1. Replace the current `monitor_commands()` function (which calls `execute_command()` directly from a background thread -- threading violation) with the CustomEvent pattern:
       - In `run()`: call `app.registerCustomEvent(CUSTOM_EVENT_ID)`, create a `CommandEventHandler(adsk.core.CustomEventHandler)` subclass, add it to the event, store reference in module-level `handlers` list to prevent GC (Pitfall 2 from research).
       - Create `MonitorThread(threading.Thread)` that only reads command files and calls `app.fireCustomEvent(CUSTOM_EVENT_ID, json.dumps({...}))` -- NO Fusion API calls in this thread.
       - In `CommandEventHandler.notify()`: parse `args.additionalInfo` as JSON, call `execute_command()`, write response file. This runs on main thread -- safe for Fusion API.
       - In `stop()`: set stop flag, remove handler, call `app.unregisterCustomEvent(CUSTOM_EVENT_ID)`.

       - Remove the `ui.messageBox` on startup (annoying for users) -- use `app.log()` instead or just print to console.

       - Import `HANDLER_MAP` from `handlers.__init__`
       - `execute_command()` does: lookup handler in HANDLER_MAP, if not found return error with list of available tools, if found call `wrap_handler(tool_name, handler, design, rootComp, params)` from helpers.errors.
       - Special case for `batch`: handle inline by iterating commands array, calling execute_command for each, stopping on first error and returning partial results per user decision.

  <action>
    **Create fusion-addin/helpers/__init__.py:**
    - Export key functions from submodules for convenient imports.

    **Create fusion-addin/helpers/errors.py (INFRA-02):**
    - `wrap_handler(tool_name, handler_fn, design, rootComp, params)`: Try/except wrapper that catches any exception and returns verbose error per user decision:
      - Format: `"{tool_name}({formatted_params}) failed: {human_readable_error}"`
      - Include a "hint" field with suggested fix (e.g., "Use get_body_info() to see available edges" for index errors, "No active sketch -- call create_sketch first" for sketch-required operations).
      - Never expose raw Python tracebacks or adsk error codes per user decision.
      - Log the full traceback to `app.log()` for debugging, but return only the user-friendly message.
    - `format_params(params)`: Format params dict as readable string for error context, e.g., `"radius=0.5, edges=[3, 7]"`.
    - `generate_hint(tool_name, params, exception)`: Map common error patterns to helpful hints. Known patterns:
      - Index out of range -> "Body has N bodies (0-{N-1}). Use get_body_info() / get_design_info()"
      - No active sketch -> "Call create_sketch() first"
      - No profiles -> "Draw geometry in sketch before extruding"
      - No bodies -> "Create geometry first (create_sketch + draw + extrude)"

    **Create fusion-addin/helpers/coordinates.py (INFRA-04):**
    - `detect_sketch_plane(sketch)`: Determine which construction plane a sketch was created on by analyzing the sketch's reference plane or transform matrix. Return 'XY', 'XZ', 'YZ', or 'custom'.
      - Primary method: Check `sketch.referencePlane` and compare against rootComp construction planes.
      - Fallback: Analyze the sketch.transform matrix -- XY has identity-like transform, XZ has rotation about X, YZ has rotation about Y.
    - `transform_to_sketch_coords(x, y, sketch)`: For XZ plane sketches, user provides (x, z_world) and we need sketch-local (x, -z_world). For XY and YZ, pass through unchanged. Per user decision: correction is silent and applies ONLY to XZ plane sketch operations.
    - `transform_from_sketch_coords(x, y, sketch)`: Reverse transform for output -- same sign flip on XZ plane. Per user decision: tool responses with coordinates are also corrected for input/output consistency.
    - `create_point(x, y, sketch=None)`: Convenience function that creates Point3D with coordinate correction applied if sketch is on XZ plane. Z coordinate is always 0 (sketch-local).

    **Create fusion-addin/helpers/selection.py (INFRA-05):**
    - `resolve_edges(body, selectors)`: Accept a list of selectors where each can be:
      - An integer (edge index) -- direct lookup via `body.edges.item(i)`
      - A string (semantic name) -- resolve via semantic_selector_map
      - Return list of BRepEdge objects. Per user decision: when a semantic selector matches multiple items, return ALL matches.
    - `resolve_faces(body, selectors)`: Same pattern for faces.
    - `label_edge(edge, index, body)`: Generate semantic label for an edge based on its geometry:
      - Compute edge midpoint relative to body bounding box
      - Determine position labels: top/bottom/left/right/front/back based on midpoint position
      - Include edge length: `"Edge 3 (top-front, linear, 5cm)"`
    - `label_face(face, index, body)`: Generate semantic label for a face:
      - Determine face type (planar, cylindrical, spherical, etc.) from `face.geometry`
      - Compute centroid relative to body bounding box for position labels
      - Include face area: `"Face 0 (top, planar, 25cm2)"`
    - `SEMANTIC_SELECTORS`: Dict mapping semantic names to resolver functions:
      - `top_face` / `bottom_face`: face with highest/lowest centroid Y (or Z depending on orientation)
      - `left_edge` / `right_edge`: edge with lowest/highest midpoint X
      - `longest_edge` / `shortest_edge`: by edge.length
      - `smallest_face` / `largest_face`: by face.area
      - `front_face` / `back_face`: by centroid Z position

    **Create fusion-addin/helpers/bodies.py:**
    - `get_body(rootComp, body_index=None)`: Get body by index, defaulting to most recent (last). Raise descriptive error if index out of range.
    - `get_sketch(rootComp, sketch_index=None)`: Get sketch by index, defaulting to most recent. Raise descriptive error if no sketches exist.
    - `get_occurrence(rootComp, index=None, name=None)`: Get occurrence by index or name. Raise descriptive error if not found.
