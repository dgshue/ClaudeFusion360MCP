  <action>
Create `fusion-addin/handlers/sketch_query.py` with one handler:

`get_sketch_info(design, rootComp, params)`:

Use standard sys.path pattern. Import label_curve from helpers.sketch_entities, get_sketch from helpers.bodies, transform_from_sketch_coords from helpers.coordinates.

The handler should return a comprehensive view of the active sketch:

1. **Curves section**: Iterate `sketch.sketchCurves` (0 to count-1). For each curve:
   - `index`: integer index
   - `label`: from label_curve(curve, index)
   - `is_construction`: curve.isConstruction
   - `is_fully_constrained`: curve.isFullyConstrained
   - `type`: class name (SketchLine, SketchCircle, etc.)

2. **Points section**: Iterate `sketch.sketchPoints` (0 to count-1). For each point:
   - `index`: integer index
   - `x`, `y`: coordinates (apply transform_from_sketch_coords for world-space output)
   - `is_origin`: True if index == 0 (first point is always the origin)
   - Note: mention in the label that point 0 is the sketch origin

3. **Constraints section**: Iterate `sketch.geometricConstraints` (0 to count-1). For each constraint:
   - `index`: integer index
   - `type`: constraint class name (e.g., "HorizontalConstraint", "PerpendicularConstraint")
   - Try to extract the constraint type from the class name, clean it up to just the type word (horizontal, vertical, etc.)

4. **Dimensions section**: Iterate `sketch.sketchDimensions` (0 to count-1). For each dimension:
   - `index`: integer index
   - `type`: dimension class name (DistanceDimension, DiameterDimension, etc.)
   - `value`: `dim.parameter.value` (the driving value)
   - `parameter_name`: `dim.parameter.name`

5. **Summary stats**:
   - `curve_count`: total curves
   - `point_count`: total points
   - `constraint_count`: total geometric constraints
   - `dimension_count`: total dimensions
   - `fully_constrained_curves`: count of curves where isFullyConstrained is True
   - `constraint_status`: "fully constrained" if all non-construction curves are constrained, "under-constrained" otherwise, "over-constrained" if any constraint errors exist

6. **Sketch metadata**:
   - `sketch_name`: sketch.name
   - `plane`: read from sketch.attributes.itemByName('FusionMCP', 'base_plane').value if it exists, else "unknown"

Return format:
```python
{
    "success": True,
    "sketch_name": "...",
    "plane": "XY",
    "curves": [...],
    "points": [...],
    "constraints": [...],
    "dimensions": [...],
    "summary": {
        "curve_count": N,
        "point_count": N,
        "constraint_count": N,
        "dimension_count": N,
        "fully_constrained_curves": N,
        "constraint_status": "under-constrained"
    }
}
```

Wrap constraint/dimension iteration in try/except for robustness -- some constraint types may have unexpected attributes. Log warnings but don't fail the whole query.

```python
@mcp.tool()
def get_sketch_info() -> dict:
    """Get detailed information about the active sketch including all curves, points, constraints, and dimensions.

    Returns curves with semantic labels (e.g., "Curve 0 (line, horizontal, 5cm)"),
    points with coordinates, geometric constraints, parametric dimensions, and
    overall constraint status.

    This is the primary tool for understanding sketch state before applying
    constraints, dimensions, or other operations. Similar to get_body_info for 3D bodies.

    Example workflow:
        1. create_sketch(plane="XY")
        2. draw_line(...), draw_circle(...)
        3. get_sketch_info()  # see all entities with indices
        4. constrain_horizontal(curve_index=0)
        5. dimension_distance(value=5, curve_index=0)
    """
    return send_fusion_command("get_sketch_info", {})
```
