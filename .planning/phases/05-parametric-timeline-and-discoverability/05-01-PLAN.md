Create `fusion-addin/handlers/parametric.py` with two handlers:

1. `create_parameter(design, rootComp, params)`:
   - Accepts: name (str, required), expression (str, required), unit (str, default "cm"), comment (str, optional)
   - Uses `design.userParameters` to check for existing parameter by name (return error if exists)
   - Creates parameter via `userParams.add(name, adsk.core.ValueInput.createByString(expression), unit, comment)`
   - Returns: `{"success": True, "parameter": name, "expression": param.expression, "value": param.value, "unit": param.unit}`

2. `set_parameter(design, rootComp, params)`:
   - Accepts: name (str, required), expression (str, required)
   - Looks up parameter via `userParams.itemByName(name)` (return error if not found)
   - Sets `param.expression = expression` (this triggers model regeneration)
   - Returns: `{"success": True, "parameter": name, "expression": param.expression, "value": param.value, "unit": param.unit}`

Create `fusion-addin/handlers/timeline.py` with four handlers and a module-level `_markers = {}` dict:

1. `get_timeline(design, rootComp, params)`:
   - Iterates `design.timeline` using `timeline.count` and `timeline.item(i)`
   - For each TimelineObject, extract: index, name, isSuppressed, isRolledBack, healthState (mapped to string via helper), isGroup
   - Map FeatureHealthStates enum values: HealthyFeatureHealthState->"healthy", WarningFeatureHealthState->"warning", ErrorFeatureHealthState->"error", SuppressedFeatureHealthState->"suppressed", RolledBackFeatureHealthState->"rolled_back"
   - Returns: `{"success": True, "marker_position": timeline.markerPosition, "count": timeline.count, "features": [...]}`

2. `edit_at_timeline(design, rootComp, params)`:
   - Accepts: position (int, required). -1 means "return to end"
   - If position == -1: call `timeline.moveToEnd()`, return success with "All features active" message
   - Validate 0 <= position <= timeline.count
   - Set `timeline.markerPosition = position`
   - Return count of suppressed features and warning: "Features after position N are now suppressed. Use edit_at_timeline(position=-1) to return to end."

3. `create_marker(design, rootComp, params)`:
   - Accepts: name (str, required)
   - Stores `_markers[name] = timeline.markerPosition`
   - Returns: `{"success": True, "marker": name, "position": position, "total_markers": len(_markers)}`

4. `undo_to_marker(design, rootComp, params)`:
   - Accepts: name (str, required)
   - If name not in `_markers`, return error with list of available markers
   - Set `timeline.markerPosition = _markers[name]`
   - Returns: `{"success": True, "marker": name, "from_position": current, "to_position": target_position}`

  <name>Task 2: Add MCP tool definitions for parametric and timeline operations</name>
  <files>mcp-server/fusion360_mcp_server.py</files>
  <action>
Add 6 new `@mcp.tool()` functions to `mcp-server/fusion360_mcp_server.py`, placed after the existing tool definitions and before `if __name__` block. Each follows the established pattern of calling `send_fusion_command()`.

Use create_sketch's docstring as the template (description + Args + Examples):

1. `create_parameter(name: str, expression: str, unit: str = "cm", comment: str = "") -> dict`:
   - Docstring: Creates a named user parameter with an expression. Expressions can reference other parameters. The parameter drives geometry when used in dimensions or features.
   - Args: name, expression (can be number "5" or reference "width * 2"), unit (cm/mm/in/ft/deg/rad), comment
   - Examples: `create_parameter(name="width", expression="5", unit="cm")`, `create_parameter(name="height", expression="width * 2", unit="cm")`
   - Body: `return send_fusion_command("create_parameter", {"name": name, "expression": expression, "unit": unit, "comment": comment})`

2. `set_parameter(name: str, expression: str) -> dict`:
   - Docstring: Updates an existing parameter's expression. The model regenerates automatically. Expressions can reference other parameters.
   - Args: name (parameter to update), expression (new value -- number or expression)
   - Examples: `set_parameter(name="width", expression="10")`, `set_parameter(name="height", expression="width + 3")`
   - Body: `return send_fusion_command("set_parameter", {"name": name, "expression": expression})`

3. `get_timeline() -> dict`:
   - Docstring: Retrieves the design timeline showing all features with their status. Use to understand design history and find positions for rollback.
   - No args
   - Examples: `get_timeline()`
   - Body: `return send_fusion_command("get_timeline", {})`

4. `edit_at_timeline(position: int) -> dict`:
   - Docstring: Moves the timeline marker to a specific position. Features after the marker are suppressed. Use position=-1 to return to end (all features active).
   - Args: position (0=before all features, N=after Nth feature, -1=end)
   - Examples: `edit_at_timeline(position=3)`, `edit_at_timeline(position=-1)`
   - Body: `return send_fusion_command("edit_at_timeline", {"position": position})`

5. `create_marker(name: str) -> dict`:
   - Docstring: Creates a named marker at the current timeline position. Markers are stored in-memory (not persistent across Fusion sessions). Use with undo_to_marker for named rollback points.
   - Args: name (marker identifier)
   - Examples: `create_marker(name="before_fillets")`, `create_marker(name="base_shape")`
   - Body: `return send_fusion_command("create_marker", {"name": name})`

6. `undo_to_marker(name: str) -> dict`:
   - Docstring: Rolls the timeline back to a previously created marker position. Features after the marker are suppressed. Use get_timeline() to see current state after rollback.
   - Args: name (marker to roll back to)
   - Examples: `undo_to_marker(name="before_fillets")`
   - Body: `return send_fusion_command("undo_to_marker", {"name": name})`

Create `fusion-addin/helpers/param_annotation.py` with a helper function:

1. `annotate_value(design, value, unit="cm")`:
   - Iterates `design.userParameters` to find any parameter whose `.value` matches the given numeric value (within a small tolerance, e.g., abs(param.value - value) < 1e-6)
   - If a matching parameter is found, return a dict: `{"value": value, "unit": unit, "driven_by": param.name, "expression": param.expression}`
   - If no match, return a simple dict: `{"value": value, "unit": unit}`
   - This is a best-effort lookup -- parameters driving dimensions through Fusion's internal parameter system may not always be detectable by value matching alone, but it covers the common case of user parameters set to explicit values.

2. `annotate_feature_dimensions(design, feature)`:
   - Inspects the feature's extent (e.g., `feature.extentOne` for extrude features) to extract the distance/angle value
   - For ExtrudeFeature: checks `feature.extentOne` -- if it's a DistanceExtentDefinition, gets `distance.value` and annotates it
   - For RevolveFeature: checks `feature.extentDefinition` angle value and annotates
   - Returns a dict of dimension annotations, e.g., `{"distance": {"value": 5.0, "unit": "cm", "driven_by": "height", "expression": "width * 2"}}` or `{"distance": {"value": 5.0, "unit": "cm"}}` if not parameter-driven
   - Wraps in try/except and returns empty dict on failure (non-breaking -- annotation is additive)
