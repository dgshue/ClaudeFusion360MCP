**mcp-server/resources/guides/coordinates.md** -- Per user decision, covers:
- All units in cm (internal Fusion unit)
- The three construction planes: XY (horizontal, default), XZ (vertical/front), YZ (vertical/side)
- XZ Y-axis inversion behavior: when sketching on XZ plane, the Y coordinate in the sketch maps to -Z in world space. The add-in's coordinate correction layer handles this transparently for construction plane sketches.
- When correction applies: XZ construction plane sketches only
- When correction does NOT apply: face sketches (face-local coordinates), 3D operations (world coordinates)
- Common mistakes with examples:
  - Expecting Y-up when drawing on XZ (it's Z-up in world, Y-up in sketch)
  - Using world coordinates in face sketches (use face-local instead)
  - Forgetting offset is in cm

**mcp-server/resources/guides/workflows.md** -- Per user decision, higher-level patterns with reasoning:
- **Box/Enclosure pattern:** sketch outer profile -> extrude -> shell (remove top face) -> fillet edges. Shell before fillet because filleting changes edge topology, making face selection harder.
- **Cylinder pattern:** sketch circle -> extrude. For hollow cylinder: sketch two concentric circles -> extrude, OR sketch circle -> extrude -> shell.
- **Sweep pattern:** sketch cross-section profile (e.g., circle for pipe) -> sketch path on different plane -> sweep profile along path. Path must be on a different sketch than the profile.
- **Loft pattern:** create profiles on different planes (using offset construction planes) -> loft between them. All profiles must be on separate planes.
- **Assembly pattern:** create components -> position with move_component -> connect with joints. Always create components before joints. Use check_interference to verify clearances.
- **Parametric pattern:** create_parameter for key dimensions -> use parameter names in expressions for extrude distances, sketch dimensions. Change one parameter -> entire design updates.
- **Gear (spur) pattern:** sketch one tooth profile using arcs/lines -> pattern_circular for all teeth -> extrude. Use construction geometry for tooth spacing calculations.

Each pattern should explain WHY the ordering matters, like a senior engineer explaining the approach.
